[ 로깅 ]
    1. 정의 : 프로그램 실행 중 발생하는 정보를 기록하는 행위

    2. 목적 : print() 대신에 로깅 사용하여 '저장' 또는 효율적인 구분(레벨) 으로 유지보수

    3. 자바 (스프링) 에서 로깅하는 방법
        권장) 롬복 에서 제공하는 @Log4j2 활용한 컨트롤러 와 서비스에서 주로 사용된다.
        @Log4j2 class Controller{}

    4. 주요 객체/함수
        log.debug()     : 개발 및 테스트 단계 에서 사용.
        log.info()      : 상태 확인 사용.
        log.warn()      : 경고 확인 사용.
        log.error()     : 예외 및 문제 사용.

    5. application.properties 파일 에서 로깅 설정
        debug < info < warn < error

        1) 스프링 에서는 info 가 기본값이다.
            logging.level.root = info
        2) 특정 패키지 에서만 설정 , 패키지명은 [java] 폴더 이하 경로이다.
            logging.level.패키지명.패키지명 = 레벨

    6. 로그 파일 저장 위치 설정
        logging.file.name = 폴더명/파일명.log
        logging.logback.rollingpolicy.max-file-size = 10MB    : 로그 파일 용량
        logging.logback.rollingpolicy.max-history = 10        : 로그 파일 최대 개수
    7. 스프링프레임워크 내부 로그 줄이기
        spring.main.log-startup-info = false
        logging.level.org.springframework.boot.web=warn
        logging.level.org.springframework.boot.web.embedded.tomcat=warn
        logging.level.org.apache.catalina=warn
        logging.level.org.apache.coyote=warn
        logging.level.org.springframework.web=warn
    8. 로그 형식
        - %msg : 로그메시지  , %n : 줄바꿈            , %d{ y M d H m s }  : 날짜/시간
        - %level : 레벨     , logger : 클래스/경로
        logging.pattern.file =
        logging.pattern.console =
