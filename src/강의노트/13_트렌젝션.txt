[ 트렌젝션 ]
    1. 데이터베이스에서 여러 연산 ( DML )을 하나의 논리적인 작업 단위
        - 이체 , 주문 , 등록 등등 , 즉] 여러 insert/update/delete/select 하나의 논리 묶음

    2. 목적 : 신뢰성 과 안전성을 보장
        *데이터 무결성 보장 : 여러 연산 중 하나라도 실패하면 전체를 실패 처리
        1) 원자성 보장 : 여러 연산이 모두 성공해야만 결과를 반영한다.
        2) 일관성 유지 : 트랜잭션 이후 데이터는 항상 일관된 상태 여야한다.
        3) 독립성/격리성 : 다른 트랜잭션과 영향을 받지 않는다.
        4) 지속성 : 트랜잭션이 성공적으로 완료되면 결과는 영구적이다.

    3. 핵심용어
        1) Transaction : 트랜잭션
        2) commit : 트랜잭션이 정상 완료 되어 최종 완료
        3) rollback : 트랜잭션 중 오류 발생 하여 이전 상태로 되돌림 뜻
        4) savepoint : 특정 시점까지의 작업을 임시저장하여 부분 롤백 하는 뜻

    4. 스프링(자바)에서 트랜잭션 적용 방법 , AOP 포함
        * 비지니스 로직(Service) 에서 주로 사용 된다.
        방법1) @Transactional void method( ){ }
        방법2) @Transactional class Service{ }

    5. 롤백 작용 기준
        * RuntimeException 실행 예외 발생 해야만 롤백이 된다.
        1) Transactional ( rollbackFor = RuntimeException.class )
        2) Transactional ( rollbackFor = 특정예외클래스명.class )
